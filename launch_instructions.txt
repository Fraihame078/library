
==========================
Инструкция по запуску проекта библиотеки
==========================

Этот документ поможет запустить локально сайт‑библиотеку, даже если вы никогда
не работали с Django. Следуя пошаговым рекомендациям ниже, вы сможете
развернуть проект, наполнить его демонстрационными данными и открыть в
браузере.

Предварительные требования
-------------------------

1. **Python 3.11+**. Скачайте и установите Python с сайта https://python.org.
2. **Пакетный менеджер pip** (обычно входит в дистрибутив Python).
3. **Git** (опционально, если вы клонируете проект с GitHub) — скачать можно
   на https://git-scm.com.

Если у вас Windows, после установки Python стоит убедиться, что команды
``python`` и ``pip`` доступны в командной строке. Откройте терминал и
выполните ``python --version`` — вы должны увидеть номер версии Python.

Скачивание проекта
------------------

1. Скачайте архив проекта (файл ``library_site_clean.zip``) с GitHub или
   иного источника и распакуйте его. Внутри будет папка ``library_site``.
2. Откройте терминал и перейдите в эту папку. Например:

   .. code-block:: bash

      cd путь/к/папке/library_site

Установка зависимостей
----------------------

Проект использует фреймворк Django и несколько сторонних библиотек. Чтобы
всё запустить, необходимо установить зависимости из файла ``requirements.txt``:

.. code-block:: bash

   pip install -r requirements.txt

В процессе pip скачает и установит Django и другие нужные пакеты.

Применение миграций
-------------------

Django хранит структуру базы данных в так называемых миграциях. Выполните
следующую команду для создания таблиц:

.. code-block:: bash

   python manage.py migrate

Загрузка демонстрационных книг
------------------------------

Чтобы библиотека не была пустой, предусмотрена команда, которая добавляет 15
книг с описаниями и обложками. Выполните:

.. code-block:: bash

   python manage.py loadsampledata

Команда проверит, есть ли книги в базе, и добавит недостающие.

Запуск сервера
--------------

Теперь можно запустить встроенный сервер Django и открыть сайт в браузере:

.. code-block:: bash

   python manage.py runserver

По умолчанию сервер будет доступен по адресу http://127.0.0.1:8000/. Откройте
этот адрес в браузере — появится главная страница библиотеки.

Учётные данные администратора
-----------------------------

В базе уже создан суперпользователь с логином ``admin`` и паролем ``admin``.
Вы можете войти в административную панель по адресу
http://127.0.0.1:8000/admin/. Через админку можно добавлять, редактировать
и удалять книги, просматривать бронирования и покупки.

Пользование сайтом
------------------

* **Главная страница** отображает список всех книг. Для VIP‑книг указана
  цена и бейдж «VIP». Обычные книги имеют счётчик доступных экземпляров.
* **Регистрация и вход**. В правом верхнем углу есть ссылки «Регистрация» и
  «Войти». Зарегистрируйтесь, чтобы бронировать или покупать книги.
* **Бронирование** доступно только для обычных книг. На странице книги
  нажмите «Зарезервировать» — она появится в разделе «Мои бронирования».
* **VIP‑книги** покупаются через кнопку «Купить». После покупки книга
  отображается в разделе «Мои покупки». Покупку VIP‑книги отменить нельзя.
* **Мои бронирования** и **Мои покупки** — ссылки в навигации, которые
  появляются после входа. В бронированиях можно вернуть книгу или отменить
  бронь.

Если при выполнении какой‑либо команды возникнут ошибки, внимательно
прочитайте текст сообщения — он обычно подсказывает, что именно нужно
исправить (например, установить Python или pip, активировать
виртуальное окружение и т. д.).

Удачного использования!
